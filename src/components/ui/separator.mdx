# Separator В· `Stable`

A thin visual divider that groups or splits content, horizontally or vertically.

## Overview

Use a Separator to draw a clear line between sections of content вЂ” a list, a toolbar, a
group of settings. Built on Radix Separator, it is `decorative` by default (hidden from
the accessibility tree). When the division carries meaning that assistive tech should
announce, set `decorative={false}` to expose the `separator` role and its
`aria-orientation`.

<Canvas of={Stories.Horizontal} />

## Anatomy

- **Rule** вЂ” the single thin line; 1px on the cross axis, stretching to fill the main
  axis (full width when `horizontal`, full height when `vertical`).

## When to use

- Use to separate sections within a card, menu, or list.
- Use a `vertical` separator to divide inline items (e.g. a breadcrumb or a meta row).
- Use `decorative={false}` when the boundary is semantically meaningful and should be
  announced.

## When not to use

- Avoid as a decorative flourish where spacing alone would group content well enough.
- Avoid for layout structure that needs a labelled region вЂ” use a landmark or heading.

## Behavior & interaction

The Separator is non-interactive вЂ” it renders no events. Its only behavioral surface is
its accessibility role, guarded by interaction tests in `separator.stories.tsx`:

- Decorative (the default) is hidden from the accessibility tree вЂ” `Decorative`.
- `decorative={false}` exposes the `separator` role вЂ” `Semantic`.
- A non-decorative `vertical` separator announces `aria-orientation="vertical"` вЂ”
  `SemanticVertical`.

## Content & UX writing

- A Separator carries no text. Keep the content it divides scannable; if a section needs
  a name, add a heading above the rule rather than relying on the line alone.

## Accessibility

- Decorative by default: rendered with `role="none"` and absent from the accessibility
  tree вЂ” verified by `Decorative`.
- With `decorative={false}` it exposes the `separator` role вЂ” verified by `Semantic`.
- A non-default (`vertical`) orientation is announced via `aria-orientation` вЂ” verified
  by `SemanticVertical`.
- axe runs on every story (`a11y: { test: 'error' }`), so role rules are gated in CI.
