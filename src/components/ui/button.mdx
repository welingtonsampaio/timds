# Button ¬∑ `Stable`

Triggers an action or event ‚Äî the primary way users commit to something in the UI.

## Overview

Use a Button to let people take an action: submit a form, confirm a choice, open a
dialog. Built on a `cva` recipe, it offers six `variant`s, six `size`s and two
`shape`s. Pass an `icon` (with `iconPlacement`) for a leading or trailing affordance;
the `loading` flag swaps that icon for a spinner and disables the control. Each view
should have a single primary Button for its main call to action.

<Canvas of={Stories.Default} />

## Anatomy

- **Container** ‚Äî the interactive surface; carries the variant styling and the hit
  target.
- **Label** ‚Äî a verb-first action ("Save", "Delete account").
- **Icon (optional)** ‚Äî a leading or trailing icon (`iconPlacement`) reinforcing the
  action.
- **Spinner** ‚Äî replaces the icon while `loading` is true.

## When to use

- Use to commit an action: submit, confirm, save, delete.
- Use the **default** variant for the single most important action in a view.
- Use **secondary** / **outline** / **ghost** for supporting actions.
- Use **destructive** for irreversible or dangerous actions.

## When not to use

- Avoid for navigation between pages ‚Äî pass `href` only when a button-styled link is
  genuinely intended.
- Avoid more than one primary Button in the same group ‚Äî it dilutes the call to action.

### Icon only
Icon-only buttons must carry an accessible name via `aria-label`. Use the size that
matches the text buttons around them: `icon` aligns with `default`, `icon-sm` with
`sm`, and `icon-lg` with `lg` ‚Äî so an icon button never looks taller or shorter than
the text button beside it.

## Behavior & interaction

These behaviors are guarded by interaction tests in `button.stories.tsx`:

- Clicking fires `onClick` exactly once ‚Äî `ClicksOnce`.
- A disabled button ignores clicks and never fires `onClick` ‚Äî `DisabledDoesNotFire`.
- While loading, the button is disabled, exposes `aria-busy` and ignores clicks ‚Äî
  `Loading`.
- `htmlType="submit"` submits the surrounding form ‚Äî `SubmitInForm`.
- `href` renders an `<a>` with the URL; a disabled link drops its `href` and leaves the
  tab order ‚Äî `LinkButton`, `LinkDisabled`.
- `asChild` merges the styling onto the child element instead of a `<button>` ‚Äî
  `AsChild`.
- The button is keyboard-focusable ‚Äî `FocusesWithKeyboard`.

## Content & UX writing

- **Labels** ‚Äî verb-first, ideally one or two words ("Save", "Add member"). Avoid vague
  labels like "OK" when a specific verb fits.
- **No terminal punctuation** in labels.
- **Icon-only** buttons must carry an accessible name via `aria-label`.

## Accessibility

- Reachable and operable by keyboard: Tab to focus, Enter/Space to activate ‚Äî verified
  by `FocusesWithKeyboard`.
- Exposes the native `button` role; `aria-busy="true"` reflects the loading state ‚Äî
  verified by `Loading`.
- A disabled link button is marked `aria-disabled` and removed from the tab order
  (`tabindex="-1"`) ‚Äî verified by `LinkDisabled`.
- axe runs on every story (`a11y: { test: 'error' }`), so contrast/name/role rules are
  gated in CI.

## Do's and Don'ts

- ‚úÖ Do use one primary Button per view for the main action.
- ‚úÖ Do show a loading state for async actions to prevent double submits.
- ‚ùå Don't disable a Button without making the reason discoverable nearby.
- ‚ùå Don't ship an icon-only Button without an `aria-label`.

## Related components & patterns

- **ButtonGroup** ‚Äî composes multiple related actions; keep at most one primary inside it.
- **Spinner** ‚Äî the loading indicator the `loading` flag renders in the icon slot.
