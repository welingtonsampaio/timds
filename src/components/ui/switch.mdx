# Switch · `Stable`

A boolean toggle that flips a single setting on or off, taking effect immediately.

## Overview

Use a Switch for an instant on/off choice — a setting that applies the moment it is
flipped, with no separate confirmation step. It is built on Radix `Switch`, so it ships
the correct `switch` role and keyboard behavior out of the box. For choices that are
submitted later with a form, prefer a **Checkbox**.

<Canvas of={Stories.Default} />

## Anatomy

- **Track** — the rounded rail that carries the chromatic `variant` and changes color
  between the on and off states.
- **Thumb** — the circle that slides across the track to indicate the state.
- **Texts (optional)** — `texts={{ on, off }}` renders a label inside the track; it is
  decorative (`aria-hidden`) and grows the rail to fit.

## When to use

- Use for a setting that takes effect immediately (notifications, dark mode, a feature
  flag).
- Use `texts={{ on, off }}` when an in-context label (ON/OFF, Ativo/Inativo) helps
  scanning a list of toggles.
- Use the `green_red` variant for a status that is meaningfully positive vs negative.

## When not to use

- Avoid for choices that only apply after submitting a form — use a **Checkbox**.
- Avoid for selecting one option among several — use a **RadioGroup**.
- Avoid relying on the inner `texts` as the only label — they are `aria-hidden`.

## Behavior & interaction

These behaviors are guarded by interaction tests in `switch.stories.tsx`:

- Clicking toggles the state and fires `onCheckedChange` with the new value —
  `TogglesOnClick`.
- The switch is keyboard-focusable and toggles with Space — `TogglesWithKeyboard`.
- A disabled switch does not toggle and does not fire `onCheckedChange` —
  `DisabledDoesNotToggle`.

## Code examples

```tsx
import { Switch } from 'timds'

// Uncontrolled
<Switch defaultChecked aria-label="Notifications" />

// Controlled
<Switch checked={enabled} onCheckedChange={setEnabled} aria-label="Dark mode" />

// With in-track labels (decorative) + an accessible name
<Switch texts={{ on: 'Ativo', off: 'Inativo' }} variant="green_red" aria-label="Status" />
```

## Content & UX writing

- **Inner texts** — keep them ultra-short (≤ 4 chars: ON/OFF, Sim/Não). The rail is
  small; longer strings make it disproportionately wide.
- **Accessible name** — describe the setting, not the action ("Dark mode", not "Toggle").
- **State labels nearby** — when you show the current state as adjacent text, keep it in
  sync with the control (see the Controlled example).

## Accessibility

- Exposes the native `switch` role with `data-state="checked" | "unchecked"`.
- Reachable and operable by keyboard: Tab to focus, Space to toggle — verified by
  `TogglesWithKeyboard`.
- The inner `texts` are `aria-hidden`; always provide an `aria-label` or an associated
  `<label htmlFor>` so the control has an accessible name.
- axe runs on every story (`a11y: { test: 'error' }`), so name/role/contrast rules are
  gated in CI.

## Do's and Don'ts

- ✅ Do give every Switch an accessible name (`aria-label` or a `<label>`).
- ✅ Do use a Switch when the change applies immediately.
- ❌ Don't use a Switch inside a form whose changes only apply on submit — use a Checkbox.
- ❌ Don't depend on the in-track `texts` as the control's only label.

## Related components & patterns

- **Checkbox** — for choices that are submitted with a form, or for multi-select lists.
- **RadioGroup** — for choosing one option among several mutually exclusive ones.
