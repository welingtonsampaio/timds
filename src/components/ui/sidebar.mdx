# Sidebar · `Stable`

A composable, collapsible application sidebar — the persistent navigation frame for a
page, with a desktop layout and a mobile off-canvas overlay built in.

## Overview

The Sidebar is not a single element but a **kit of building blocks** wired together by a
`SidebarProvider` context: a header, a scrollable content area of groups and menus, a
footer, and a rail. It collapses to an icon rail or slides off-canvas, remembers its state
in a cookie, and toggles via a trigger, the rail, or `Cmd/Ctrl + B`. On small screens it
becomes an off-canvas overlay automatically. Wrap your whole layout in `SidebarProvider`,
put the navigation in `Sidebar`, and render the page beside it in `SidebarInset`.

- **`SidebarProvider`** — the context owner. Wrap the entire layout; it holds open/collapsed
  state, the mobile flag, the `Cmd/Ctrl + B` shortcut, and the cookie persistence. Read it
  with `useSidebar()`.
- **`Sidebar`** — the navigation container. Props: `side` (`left`/`right`), `variant`
  (`sidebar`/`floating`/`inset`), `collapsible` (`offcanvas`/`icon`/`none`).
- **`SidebarHeader`** / **`SidebarFooter`** — pinned top and bottom regions (brand switcher,
  user menu).
- **`SidebarContent`** — the scrollable middle; holds one or more groups.
- **`SidebarGroup`** + **`SidebarGroupLabel`** + **`SidebarGroupContent`** + **`SidebarGroupAction`**
  — a titled section, with an optional action button on the label.
- **`SidebarMenu`** → **`SidebarMenuItem`** → **`SidebarMenuButton`** — the list, its rows, and
  the interactive row control (`isActive`, `variant`, `size`, `tooltip`).
- **`SidebarMenuAction`** / **`SidebarMenuBadge`** — a per-row action button and a count badge.
- **`SidebarMenuSub`** → **`SidebarMenuSubItem`** → **`SidebarMenuSubButton`** — a nested level
  of links.
- **`SidebarMenuSkeleton`** — a loading placeholder shaped like a menu row.
- **`SidebarRail`** — the thin edge strip that toggles the sidebar.
- **`SidebarTrigger`** — the toggle button, placed in your page header.
- **`SidebarInset`** — the main content area beside the sidebar; pair with `variant="inset"`.

## When to use

- Use as the primary, persistent navigation for an application shell or dashboard.
- Use when navigation must stay reachable while the user works in the main content.
- Use `collapsible="icon"` when screen space is tight but the nav should remain visible.

## When not to use

- Avoid for transient, on-demand menus tied to one control — use a **DropdownMenu**.
- Avoid for a temporary panel that overlays content briefly — use a **Drawer** or **Dialog**.
- Avoid for in-page section switching — use **Tabs**.

## Behavior & interaction

- `SidebarTrigger` toggles the sidebar, flipping its `data-state` between `expanded` and
  `collapsed` — see `TogglesViaTrigger`.
- `Cmd/Ctrl + B` toggles it from anywhere on the page — see `TogglesViaKeyboardShortcut`.
- `SidebarRail` (the edge strip) toggles it too, and shares the "Toggle Sidebar" label.
- When collapsed to icons, a `SidebarMenuButton` with a `tooltip` reveals its label on
  hover or focus — see `CollapsedShowsTooltip`.
- Navigation rows are buttons; submenu rows are links — see `RendersNavigation`.
- Below the `md` breakpoint the sidebar renders as an off-canvas overlay (an internal
  Sheet); the same trigger opens it. State persists across reloads via a cookie.

## Code examples

Minimal application shell:

```tsx
import {
  Sidebar,
  SidebarContent,
  SidebarGroup,
  SidebarGroupContent,
  SidebarGroupLabel,
  SidebarInset,
  SidebarMenu,
  SidebarMenuButton,
  SidebarMenuItem,
  SidebarProvider,
  SidebarTrigger,
} from 'timds'

export function AppShell({ children }: { children: React.ReactNode }) {
  return (
    <SidebarProvider>
      <Sidebar>
        <SidebarContent>
          <SidebarGroup>
            <SidebarGroupLabel>Platform</SidebarGroupLabel>
            <SidebarGroupContent>
              <SidebarMenu>
                <SidebarMenuItem>
                  <SidebarMenuButton isActive tooltip="Home">
                    <HomeIcon />
                    <span>Home</span>
                  </SidebarMenuButton>
                </SidebarMenuItem>
              </SidebarMenu>
            </SidebarGroupContent>
          </SidebarGroup>
        </SidebarContent>
      </Sidebar>
      <SidebarInset>
        <header className="flex h-12 items-center gap-2 border-b px-4">
          <SidebarTrigger />
        </header>
        {children}
      </SidebarInset>
    </SidebarProvider>
  )
}
```

Driving it from your own UI with `useSidebar()`:

```tsx
import { useSidebar } from 'timds'

function CollapseButton() {
  const { state, toggleSidebar } = useSidebar()
  return <button onClick={toggleSidebar}>{state === 'expanded' ? 'Collapse' : 'Expand'}</button>
}
```

Controlled open state (`SidebarProvider` is controllable; default is `defaultOpen`):

```tsx
const [open, setOpen] = useState(true)

<SidebarProvider open={open} onOpenChange={setOpen}>
  {/* … */}
</SidebarProvider>
```

## Content & UX writing

- Keep menu labels short and noun-based (`Inbox`, `Settings`), title-case, one or two words.
- Group related destinations under a `SidebarGroupLabel`; keep labels to a single line.
- Always pair an icon with a text label; the icon alone is only acceptable in the collapsed
  icon rail, where the `tooltip` supplies the name.
- Reserve `SidebarMenuButton variant="outline"` and badges for signal (counts, status), not
  decoration.

## Accessibility

- Operable by keyboard: every `SidebarMenuButton`/`SidebarMenuSubButton` is focusable and
  activates with Enter/Space (native button/link semantics).
- `SidebarTrigger` and `SidebarRail` are real buttons with the accessible name
  "Toggle Sidebar"; the `Cmd/Ctrl + B` shortcut mirrors them — see
  `TogglesViaKeyboardShortcut`.
- In the icon-collapsed state, the `tooltip` on a `SidebarMenuButton` restores the visible
  label and opens on focus, not only hover — see `CollapsedShowsTooltip`.
- Give icon-only controls (`SidebarMenuAction`, `SidebarGroupAction`) an explicit
  `aria-label`.
- The active item carries `data-active="true"`; pair it with `aria-current="page"` on your
  link for assistive tech — see `RendersNavigation`.
- Axe runs on every story (`a11y: { test: 'error' }`), gating static a11y rules.

## Do's and Don'ts

- ✅ Do wrap the whole layout in `SidebarProvider` and place page content in `SidebarInset`.
- ✅ Do supply a `tooltip` on menu buttons when using `collapsible="icon"`.
- ✅ Do label icon-only actions and mark the active route with `aria-current`.
- ❌ Don't render a `Sidebar` (or `useSidebar()`) outside `SidebarProvider` — the hook throws.
- ❌ Don't nest more than one sublevel of `SidebarMenuSub`; flatten deep trees instead.
- ❌ Don't use the Sidebar for transient menus — reach for **DropdownMenu** or **Drawer**.

## Related components & patterns

- **DropdownMenu** — on-demand action menus, including the user menu often placed in
  `SidebarFooter`.
- **Drawer** / **Dialog** — temporary overlays, as opposed to persistent navigation.
- **Tooltip** — supplies labels for collapsed menu buttons (mounted by the provider).
- **Skeleton** — `SidebarMenuSkeleton` is built from it for loading rows.
- **Separator** — `SidebarSeparator` divides regions inside the sidebar.
