# Textarea ¬∑ `Stable`

A multi-line text field for free-form input that can span several lines.

## Overview

Use a Textarea when input is naturally longer than a single line ‚Äî a comment, a bio, a
message. It is a thin wrapper over the native `<textarea>` sharing the same tokens as
**Input**, so the two line up in a form. It auto-grows with its content
(`field-sizing-content`) from a minimum height, and supports the standard `disabled` and
`aria-invalid` states.

<Canvas of={Stories.Default} />

## Anatomy

- **Field** ‚Äî the bordered, resizable text area; carries focus, disabled and invalid
  styling.
- **Placeholder (optional)** ‚Äî hint text shown while the field is empty; not a substitute
  for a label.
- **Label (external)** ‚Äî an associated `<label htmlFor>` providing the accessible name.

## When to use

- Use for free-form text that may wrap to multiple lines (messages, notes, descriptions).
- Use when you want the field to grow with its content rather than scroll.

## When not to use

- Avoid for single-line values (name, email, search) ‚Äî use **Input**.
- Avoid for choosing from fixed options ‚Äî use **Select** or **RadioGroup**.

## Behavior & interaction

These behaviors are guarded by interaction tests in `textarea.stories.tsx`:

- Typing fires `onChange` and preserves line breaks ‚Äî `TypesText`.
- The field is keyboard-focusable via Tab ‚Äî `FocusesWithKeyboard`.
- A disabled field is not editable and does not fire `onChange` ‚Äî `DisabledDoesNotType`.
- The invalid state exposes `aria-invalid="true"` ‚Äî `InvalidExposesAria`.

## Content & UX writing

- **Label** ‚Äî always provide one; describe the content expected ("Bio", "Message").
- **Placeholder** ‚Äî an example or hint, never a replacement for the label.
- **Error messages** ‚Äî specific and actionable; associate them with `aria-describedby`.

## Accessibility

- Exposes the native `textbox` role; give it an accessible name via an associated
  `<label htmlFor>` or `aria-label`.
- Reachable and operable by keyboard: Tab to focus, type to edit ‚Äî verified by
  `FocusesWithKeyboard` and `TypesText`.
- The error state sets `aria-invalid="true"` ‚Äî verified by `InvalidExposesAria`; pair it
  with `aria-describedby` pointing at the error text.
- axe runs on every story (`a11y: { test: 'error' }`), so name/role/contrast rules are
  gated in CI.

## Do's and Don'ts

- ‚úÖ Do give every Textarea an associated label.
- ‚úÖ Do set `aria-invalid` and link an error message via `aria-describedby` on failure.
- ‚ùå Don't use a placeholder as the only label.
- ‚ùå Don't use a Textarea for single-line input ‚Äî use Input.

## Related components & patterns

- **Input** ‚Äî for single-line values; shares the same tokens and form layout.
- **InputGroup** ‚Äî for affixing icons/addons around a field; see `InputGroupTextarea`
  for a Textarea inside a group.
