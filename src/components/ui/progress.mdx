# Progress Â· `Stable`

A linear bar that communicates how far along a determinate task is, or that work of
unknown duration is underway.

## Overview

Use a Progress bar to give feedback on an operation that takes time â€” an upload, an
import, a multi-step form. Pass a `value` from 0 to 100 for a determinate bar; omit it for
the indeterminate state when the remaining time is unknown. It is built on Radix
`Progress`, so it exposes the `progressbar` role and the matching ARIA value attributes.

## Anatomy

- **Track** â€” the rounded, translucent background that defines the bar's full length.
- **Indicator** â€” the solid fill whose width reflects the current `value`.

## When to use

- Use to show determinate progress of a task with a measurable percentage (upload, sync).
- Use the indeterminate state when work is ongoing but its duration is unknown.
- Use a semantic `variant` to tie the bar to an outcome (e.g. `destructive` for a quota
  near its limit).

## When not to use

- Avoid for a brief, indeterminate wait with no measurable progress â€” use a **Spinner**.
- Avoid as a step indicator for a wizard â€” use a dedicated stepper pattern.
- Avoid for storage/segmented meters that need multiple values â€” this bar is single-value.

## Behavior & interaction

Progress is non-interactive; its contract is the ARIA semantics, guarded by interaction
tests in `progress.stories.tsx`:

- Exposes the `progressbar` role with `aria-valuemin` / `aria-valuemax` / `aria-valuenow`
  and a `loading` state â€” `HasProgressbarSemantics`.
- Without `value`, Radix marks it `indeterminate` and omits `aria-valuenow` â€”
  `IndeterminateHasNoValueNow`.
- At `value={100}` it reports the maximum and the `complete` state â€” `CompleteReflectsMax`.

- **Accessible name** â€” describe the task, not the widget ("Uploading files", not

- **Pair with text** â€” when the percentage matters to the user, show it next to the bar;

- **Indeterminate copy** â€” accompany an indeterminate bar with a short status like

## Accessibility

- Exposes the `progressbar` role with `aria-valuemin="0"`, `aria-valuemax="100"` and, when
  determinate, `aria-valuenow` â€” verified by `HasProgressbarSemantics`.
- The indeterminate state omits `aria-valuenow` so assistive tech announces unknown
  progress â€” verified by `IndeterminateHasNoValueNow`.
- Always provide an accessible name via `aria-label` or `aria-labelledby`; axe runs on
  every story (`a11y: { test: 'error' }`), so a nameless bar fails CI.
