# Examples

Real pages composed entirely from timds components — no one-off CSS, no raw colors.
They double as a **cookbook** (copy a story, swap the data) and as a **stress test** for
the design system: every example renders in a real browser and passes the same
accessibility checks as the component gallery.

They're ordered from the simplest single-card screen to a full application shell. Each
one is also an interaction-tested story — open the **Interactions** panel to watch its
`play` function exercise the page.

## How to read them

- **Tokens only.** Colors come from semantic utilities (`bg-primary`,
  `text-muted-foreground`, `text-success-text`) — flip the **Theme** toolbar and every
  page re-themes itself.
- **Accessible by construction.** Inputs are labelled, icon-only buttons are named, and
  contrast passes AA. The same rules are documented on the **Accessibility** page.
- **Composition first.** Notice how little glue code each page needs: the components do
  the heavy lifting, and layout is plain Tailwind fl/grid.

> Want a new building block instead of a page? Browse the component categories —
> _Layout_, _Navigation_, _Data Entry_, _Data Display_, _Feedback_ and _Overlays_.
