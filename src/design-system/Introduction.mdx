# Timds — This Is My Design System

> **timds** (pronounced "tim-dis") stands for _This Is My Design System_ — a personal
> project to build a reusable component library and a cohesive design language for my
> apps, so every product looks consistent and ships faster.

A React component library with a **built-in design system**, powered by
**shadcn/ui (new-york)** and **Tailwind CSS v4**. Tokens are defined once as CSS
variables in `src/styles.css` and consumed everywhere through Tailwind's
`@theme inline` bridge — so re-theming the whole library is a single-file change.

The visual language is derived from a dark, modern **analytics dashboard**: deep
indigo surfaces, a vibrant violet brand color, and a cyan support accent.

## Design principles

These are the rules every component and example in this library follows.

<table>
  <thead>
    <tr>
      <th>Principle</th>
      <th>What it means in practice</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>Tokens, never raw colors</strong></td>
      <td>
        Components reference semantic utilities (<code>bg-primary</code>,
        <code>text-muted-foreground</code>) — never hex or <code>oklch</code> values.
        One token file drives both themes.
      </td>
    </tr>
    <tr>
      <td><strong>Accessible by default</strong></td>
      <td>
        Every interactive state is keyboard-reachable, labelled, and passes WCAG AA
        contrast. Accessibility is enforced in CI, not reviewed by eye — see the
        <strong>Accessibility</strong> page.
      </td>
    </tr>
    <tr>
      <td><strong>Composition over configuration</strong></td>
      <td>
        Prefer small primitives you assemble (<code>Card</code> + <code>Badge</code> +
        <code>Button</code>) over monolithic components with dozens of props. The
        <strong>Examples</strong> show this end to end.
      </td>
    </tr>
    <tr>
      <td><strong>Dark-first, light-ready</strong></td>
      <td>
        The palette is tuned against the dark reference, with a light mode that mirrors
        it. Toggle the <strong>Theme</strong> switch in the toolbar to compare.
      </td>
    </tr>
    <tr>
      <td><strong>Docs are tests</strong></td>
      <td>
        Each story renders in a real browser as a smoke + accessibility test; many also
        carry interaction (<code>play</code>) assertions. A broken example fails the build.
      </td>
    </tr>
  </tbody>
</table>

## How tokens flow

<ol style={{ listStyleType: 'decimal' }}>
  <li>
    <strong><code>:root</code> / <code>.dark</code></strong> in <code>styles.css</code>
    declare the raw <code>oklch</code> values.
  </li>
  <li>
    <strong><code>@theme inline</code></strong> maps each <code>--token</code> to a
    Tailwind color utility (e.g. <code>--color-primary</code> → <code>bg-primary</code>,
    <code>text-primary</code>).
  </li>
  <li>
    <strong>Components</strong> reference only the semantic utilities — never raw colors —
    so re-theming is a single-file change.
  </li>
</ol>

See the **Theming** page for the full pipeline, dark mode, and how to customize tokens
in a consuming app.

## Using the library

The library is distributed **directly from GitHub** (no npm registry). On install, the
`prepare` script builds `dist/`, so consumers import components and the precompiled
tokens straight away:

```tsx
import { Button, Card, CardHeader, CardTitle, Badge } from 'timds'
import 'timds/styles.css'

export function Example() {
  return (
    <Card>
      <CardHeader>
        <CardTitle>Revenue</CardTitle>
        <Badge variant="success">+9.8%</Badge>
      </CardHeader>
    </Card>
  )
}
```
