# 0001 — Build as a library with Vite

- Status: Accepted
- Date: 2026-06-17

## Context

The library needs to be packaged for consumption by React applications: generating
a JavaScript bundle, type declarations (`.d.ts`), and the design system's CSS.
We need a build tool that handles React, TypeScript, and CSS (Tailwind) well and
that provides an agile development server for the playground.

## Decision

Use **Vite in library mode** (`build.lib`) as the build tool:

- Output in **ESM** (`formats: ['es']`), with a single entrypoint at `src/index.ts`.
- `react`, `react-dom`, and the JSX runtimes marked as **external** (they are not
  bundled — they come from the consumer via peer dependencies).
- The CSS is extracted into a single file (`dist/timds.css`).
- Types generated by `vite-plugin-dts`.

## Alternatives considered

- **tsup** — great for JS/TS libs, but its handling of CSS + Tailwind v4 and the
  integration with the ecosystem (playground, Storybook) is less straightforward.
- **Plain Rollup** — more manual configuration; Vite already wraps Rollup.
- **tsc only** — does not handle CSS bundling nor provide a dev server.

## Consequences

- **Positive:** a single toolchain covers the library build, the dev server
  (playground), and is reused by Storybook and Vitest.
- **Negative/limitations:**
  - We only generate ESM. Consumers using pure CommonJS are not supported
    (accepted: the target is modern React, which uses ESM/bundlers).
  - The `.d.ts` files are emitted preserving the `src/` structure (under
    `dist/src/`) because of the relative imports between them; for this reason the
    `types` field in `package.json` points to `./dist/src/index.d.ts`.
